### 스프링 계층 구조

- Presentation Layer
    - 웹 or 전통적인 UI 기술 담당(MVC)
- Business Layer
    - 전체 애플리케이션의 비즈니스 로직
- Persistence Layer
    - DB 엑세스

### 서비스 추상화

- `@Service` 관계 X
- 서비스는 일반적인 용어 - 다양한 의미
    - 클라이언트에게 서비스를 제공하는 오브젝트(서버)
    - 일반적으로 상태를 가지지 않는다.(스프링 빈)
        - 상태는 DB에 저장
- 서비스의 종류
    - 애플리케이션 서비스
        - 애플리케이션 계층에 존재하는 서비스 - `@Service` 붙는다.
        - 비즈니스 로직 시작 / 종료가 되는 경계
    - 도메인 서비스
        - 도메인 모델 패턴에서 특정 엔티티로 표현하기 어려운 비즈니스 로직 관리
    - 인프라 서비스
        - 서비스 추상화의 대상
        - 기술을 제공하는 서비스(기술적인 관심사: 이메일, 트랜잭션 ..)
